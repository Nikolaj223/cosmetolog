# Beauty Lab Demo

Учебный сайт косметологической клиники «Лаборатория красоты», собранный на `HTML`, `CSS` и `vanilla JS`.

## Что внутри

- Публичный сайт с блоками услуг, специалистов, портфолио, отзывов, акций и контактов.
- Многошаговая онлайн-запись с выбором услуги, специалиста, даты и времени.
- Админ-панель с локальной авторизацией, редактированием контента, модерацией отзывов, управлением расписанием и экспортом записей в CSV.
- Локальное хранилище на базе `localStorage`, чтобы без сервера показать сценарий, близкий к БД.

## Почему структура стала лучше

Наставник справедливо обратил внимание, что слишком тяжёлый `index.html` хуже читать и поддерживать. Поэтому главная страница теперь устроена так:

- `index.html` стал лёгкой точкой входа.
- Основная разметка главной вынесена в отдельную оболочку `assets/js/home-shell.js`.
- Данные сайта лежат отдельно от логики.
- Хранилище лежит отдельно от интерфейса.

Это не магически ускоряет сайт само по себе, но делает проект чище, проще для ревью и проще для дальнейшего роста.

## Как запустить

1. Можно открыть [index.html](c:/da/index.html) напрямую в браузере.
2. Если браузер ограничивает `localStorage` для `file://`, запустите локальный сервер:

```powershell
powershell -ExecutionPolicy Bypass -File .\serve.ps1
```

После этого откройте `http://localhost:8080`.

## Подробная структура проекта

### Корень проекта

- [index.html](c:/da/index.html)
  Главная точка входа публичного сайта.
  Сейчас внутри только базовая HTML-оболочка, подключение стилей и скриптов.

- [admin.html](c:/da/admin.html)
  Отдельная страница административной панели.
  Здесь хранится разметка админки: логин, меню разделов, формы управления.

- [help.html](c:/da/help.html)
  Страница помощи для посетителя.
  Нужна как упрощённое пользовательское руководство.

- [privacy.html](c:/da/privacy.html)
  Короткая политика обработки персональных данных.

- [serve.ps1](c:/da/serve.ps1)
  Простой локальный сервер на PowerShell.
  Нужен, если сайт плохо работает при открытии напрямую через `file://`.

- [README.md](c:/da/README.md)
  Этот файл.
  Здесь нужно держать описание проекта, запуск и карту структуры.

### Папка `assets`

- `assets` нужна для всего, что не является корневыми HTML-страницами.
- Внутри три основные части: `styles`, `js`, `media`.

### Папка `assets/styles`

- [assets/styles/main.css](c:/da/assets/styles/main.css)
  Главные стили публичной части сайта.
  Здесь лежат:
  цвета, сетки, карточки, модалки, кнопки, hero-блок, footer, mobile media queries.

- [assets/styles/admin.css](c:/da/assets/styles/admin.css)
  Стили только для админки.
  Здесь лежат:
  боковая панель, карточка авторизации, таблица записей, формы администрирования.

### Папка `assets/js`

- [assets/js/data.js](c:/da/assets/js/data.js)
  Начальные демо-данные сайта.
  Здесь лежит почти вся “база” по умолчанию:
  услуги, категории, специалисты, отзывы, акции, расписания, записи, контакты, hero-контент.

- [assets/js/store.js](c:/da/assets/js/store.js)
  Локальное хранилище.
  Отвечает за:
  чтение из `localStorage`, запись в `localStorage`, сброс демо-данных, обновление коллекций.

- [assets/js/home-shell.js](c:/da/assets/js/home-shell.js)
  Оболочка главной страницы.
  Здесь хранится большая разметка публичной страницы, которая раньше лежала прямо в `index.html`.
  То есть:
  header, hero, секции, footer, модалки и контейнеры под динамический контент.

- [assets/js/home-helpers.js](c:/da/assets/js/home-helpers.js)
  Общие вспомогательные функции главной страницы.
  Здесь лежат:
  экранирование HTML, форматирование дат, пустые состояния, звёзды рейтинга, базовое состояние мастера записи.

- [assets/js/app.js](c:/da/assets/js/app.js)
  Поведение публичного сайта.
  Отвечает за:
  рендер карточек, фильтры, модалки, отзывы, онлайн-запись, работу кнопок, обновление экрана.

- [assets/js/admin.js](c:/da/assets/js/admin.js)
  Поведение админки.
  Отвечает за:
  авторизацию, CRUD-операции, модерацию отзывов, статусы записей, экспорт CSV, редактирование контента.

### Папка `assets/media`

- [assets/media/logo.svg](c:/da/assets/media/logo.svg)
  Логотип сайта.

- `assets/media/portraits`
  SVG-портреты специалистов.

- `assets/media/portfolio`
  SVG-иллюстрации кейсов до/после.

## Если нужно что-то изменить

### Если нужно поменять текст или демо-данные

- Иди в [assets/js/data.js](c:/da/assets/js/data.js)
- Там меняются:
  названия услуг, цены, специалисты, акции, отзывы, контакты, hero-тексты

### Если нужно поменять внешний вид

- Публичная часть: [assets/styles/main.css](c:/da/assets/styles/main.css)
- Админка: [assets/styles/admin.css](c:/da/assets/styles/admin.css)

### Если нужно поменять логику главной страницы

- Иди в [assets/js/app.js](c:/da/assets/js/app.js)
- Там меняются:
  фильтры, модалки, запись, обработка форм, рендер блоков

### Если нужно поменять структуру блоков главной страницы

- Иди в [assets/js/home-shell.js](c:/da/assets/js/home-shell.js)
- Там меняется:
  порядок секций, разметка hero, footer, модалки и другие крупные куски страницы

### Если нужно поменять админку

- Разметка: [admin.html](c:/da/admin.html)
- Логика: [assets/js/admin.js](c:/da/assets/js/admin.js)
- Стили: [assets/styles/admin.css](c:/da/assets/styles/admin.css)

## Важное допущение

По ТЗ предполагается `SQLite` и серверные уведомления, но в текущем окружении нет `Node.js`. Поэтому для рабочей демо-версии запись клиентов и управление контентом реализованы на чистом фронтенде через `localStorage`. Структура приложения сделана так, чтобы позже можно было заменить слой хранения на API и реальную БД.
